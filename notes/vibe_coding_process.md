# Vibe Coding 제작 과정 기록

## 1. 처음 만들고 싶었던 것
13주차 TODO에서는 노래방 점수를 저장하고 곡별 최고점, 평균점, 최근 변화 등을 분석하는 프로그램을 만들고 싶다고 작성했다.

## 2. 첫 구현 방향
처음에는 간단한 입력/조회 프로그램을 생각했지만, 최종 구현과제에서는 사용자가 직접 체험할 수 있는 결과물이 중요하다고 판단하여 Flask 기반 웹앱으로 확장했다.

## 3. 막혔던 부분
- 같은 곡을 여러 번 입력했을 때 곡별로 묶어서 계산하는 부분
- 최근 점수 변화가 단순 최고점보다 더 의미 있게 보이도록 만드는 부분
- 외부 배포 시 `127.0.0.1`만 바라보면 접속이 되지 않는 문제
- 그래프를 넣고 싶었지만 외부 라이브러리에 의존하면 실행 환경에 따라 깨질 수 있는 문제

## 4. 해결 방법
- 리스트와 딕셔너리를 이용해 곡명 기준으로 데이터를 묶었다.
- 최근 3회 평균과 이전 평균을 비교하여 상승세/하락세/유지를 판단했다.
- 표준편차를 이용해 점수 안정도 지수를 만들었다.
- `host="0.0.0.0"`, `PORT` 환경변수, gunicorn, Procfile을 추가해 Render/Replit 배포가 가능하게 했다.
- SVG polyline을 이용해 외부 JS 라이브러리 없이 그래프를 구현했다.

## 5. 결과적으로 배운 점
프로그래밍은 문법을 아는 것보다 실제 사용자가 쓸 수 있는 흐름으로 만드는 것이 더 어렵다는 것을 느꼈다. 특히 입력, 저장, 분석, 화면 표시, 배포를 연결하면서 하나의 작은 서비스가 만들어지는 과정을 경험했다.
